## Риски и mitigation

### Риск 1: Сложность динамической перебалансировки шардов
- **Mitigation:** Использовать готовые паттерны (consistent hashing), начать с простой round-robin балансировки

### Риск 2: Производительность sliding window при большой нагрузке
- **Mitigation:** Использовать ring buffer, ограничить размер окна, протестировать с нагрузкой

### Риск 3: Проблемы с CGO при интеграции Rust
- **Mitigation:** Протестировать на целевой платформе, иметь fallback на pure Go валидацию

### Риск 4: Время на настройку HPA с custom metrics
- **Mitigation:** Начать с простого CPU-based autoscaling, добавить queue metrics позже
