# План исправления критических проблем для сдачи задания

**Дата создания:** 2026-05-26  
**Приоритет:** Высокий  
**Статус:** В разработке

## Критические проблемы (по замечаниям преподавателя)

### 1. Нет полноценного рабочего pipeline
**Проблема:** Компоненты существуют отдельно, но не показан полный рабочий поток:
```
Эмулятор → Go Collector → Kafka → Python Processor → БД → Dashboard
```

**Что нужно исправить:**
- Собрать и продемонстрировать единый pipeline
- Обеспечить сквозную передачу данных через все компоненты

### 2. Нет Kafka producer в Go
**Проблема:** В задании требуется Go-сборщик пишет данные в Kafka. Kafka consumer есть, но producer либо отсутствует, либо не интегрирован.

**Что нужно исправить:**
- Добавить Kafka producer в Go collector
- Реализовать отправку агрегированных данных в topic
- Добавить retry и reconnect логику

### 3. Не реализовано sliding window
**Проблема:** Требуется скользящее окно 5 минут. В проекте реализован только tumbling window.

**Что нужно исправить:**
- Добавить sliding window (окно 5 минут, обновление каждые N секунд)
- Реализовать хранение состояния окна

### 4. Нет benchmark результатов Go vs Python
**Проблема:** В задании требуется сравнение скорости, CPU, памяти, throughput.

**Что нужно исправить:**
- Подготовить нагрузочный тест
- Измерить requests/sec, RAM, CPU
- Оформить результаты сравнения

### 5. Kubernetes развёртывание неполное
**Проблема:** Есть отдельные YAML, но нет полноценного deployment. Отсутствуют:
- etcd deployment
- Kafka deployment  
- PostgreSQL deployment
- namespace
- ConfigMap/Secrets
- HPA не заработает без custom metrics

**Что нужно исправить:**
- Добавить полный docker-compose или k8s stack
- Создать namespace
- Добавить kafka + postgres manifests
- Реализовать рабочие readiness/liveness endpoints

### 6. Apache Arrow не интегрирован в pipeline
**Проблема:** Arrow Flight реализован отдельно как demo. Нет реальной передачи:
```
Aggregator → Arrow Flight → Python
```

**Что нужно исправить:**
- Подключить Arrow к реальному потоку данных
- Передавать агрегаты через Arrow

### 7. Dashboard не realtime
**Проблема:** Dashboard работает через polling БД. Но по заданию требуется realtime обновление.

**Что нужно исправить:**
- Реализовать auto-refresh каждые 1–2 секунды
- Или добавить WebSocket для push-обновлений

## Риски и mitigation

### Риск 1: Сложность интеграции Kafka
- **Mitigation:** Использовать готовые библиотеки (confluent-kafka-go), начать с простого producer

### Риск 2: Производительность sliding window
- **Mitigation:** Использовать эффективные структуры данных (кольцевой буфер), протестировать с нагрузкой

### Риск 3: Время на настройку k8s
- **Mitigation:** Использовать minikube и готовые helm charts для Kafka/PostgreSQL

### Риск 4: Нехватка времени
- **Mitigation:** Сфокусироваться на критических проблемах (pipeline, Kafka, sliding window)
